# 全国村界数据集 / China Village Boundaries Dataset

## 概述

中国全国范围的村级行政边界空间数据（Shapefile 格式），覆盖 33 个省级行政区划单位，包含约 87.5 万条村级边界记录。

## 来源与数据说明

- **原始分享者**：[@elpwc](https://x.com/elpwc/status/2064590494974591226)（通过[网盘分享](https://x.com/elpwc/status/2064590494974591226)）
- **整理/再上传者**：[@thedavidweng](https://x.com/thedavidweng)
- **数据时间跨度**：据原分享者估计，数据来自 2010–2022 年间，大部分约在 2016 年左右
- **数据来源**：网络收集整理，原始出处不详。推测部分数据来源于国土三调（第三次全国国土调查）及相关政府公开数据

## 数据格式

- **主要格式**：ESRI Shapefile（.shp / .shx / .dbf / .prj / .cpg）
- **辅助格式**：FileGDB（.gdb）、GeoJSON、KML、GML（部分省份）
- **编码**：UTF-8（大部分）或 GBK/CP936（部分省份，见下方说明）
- **坐标参考系（CRS）**：不统一，见下方详细说明

## 坐标参考系（CRS）

**已统一为 WGS84 (EPSG:4326)。** 所有 58 个 .shp 文件的坐标值均为地理坐标（度），.prj 文件已声明 WGS84。

> 原始 .prj 文件曾声明多种 CRS（CGCS2000、Krasovsky_1940_Albers、Hong_Kong_1980_Grid 等），但 .shp 坐标值始终为地理坐标。详见 CHANGELOG.md。

## 字段编码说明

所有 58 个 .shp 的 .cpg 文件均声明 **UTF-8** 编码。大部分数据集的字段名和字段值都是 UTF-8，但存在以下已知问题：

**字段值为 GBK 的数据集**（.cpg 声明 UTF-8，但字段值实际为 GBK 编码）：
- 河北村界、河南村界：字段名 UTF-8 正常，字段值（村名等中文内容）为 GBK
- 陕西村界（4 个数据集）：同上

> GIS 软件读取时，字段名显示正常，但中文字段值可能出现乱码。手动处理时需将字段值按 GBK 解码。

**UTF-8 字段名被 DBF 11 字节限制截断**（多个省份）：
- 广东全省、上海 2 套、山西、四川、甘肃、浙江、福建、黑龙江等省份的部分 .dbf 中，UTF-8 中文字段名因 DBF 格式 11 字节限制被截断
- 完整字段名已通过字节分析恢复，见 CHANGELOG.md

## 字段 Schema

各省 .dbf 的字段结构不统一。完整的字段映射表见 **[SCHEMA_MAPPING.md](SCHEMA_MAPPING.md)**（含 dry run 验证结果）。

映射覆盖 11 个核心管理层级字段：

| 标准字段 | 说明 | 覆盖率 |
|---|---|---|
| `province_code` / `province_name` | 省级 | 100% |
| `city_code` / `city_name` | 地市级 | 100% |
| `county_code` / `county_name` | 区县级 | 100% |
| `township_code` / `township_name` | 乡镇级 | 100% |
| `village_code` / `village_name` | 村级 | 100% |
| `note` | 备注 | 100% |

> 52/58 数据集的所有字段完全映射（90%），6 个含 DBF 11 字节截断的中文字段名（上海 2 套数据的社区管理补充字段）。截断字段的完整名称已通过字节分析和数据值验证恢复，见 CHANGELOG.md。

常见字段模式（原始名称）：

| 字段类型 | 常见原始名称 | 说明 |
|---|---|---|
| 行政区划代码 | `XZQDM`, `xzqdm`, `AREA_CODE`, `PAC`, `code` | 12 位或 9 位行政区划代码 |
| 行政区划名称 | `XZQMC`, `NAME`, `name`, `村名称`, `村级名` | 村/社区名称 |
| 层级代码 | `省级码`, `市代码`, `区县码`, `乡镇码`, `村级码` | 各级行政区划代码（部分省份拆分存储） |
| 层级名称 | `省级名`, `市名`, `县区名`, `乡镇名`, `村名` | 各级行政区划名称 |
| 几何属性 | `SHAPE_Leng`, `SHAPE_Area` | 周长和面积（单位取决于 CRS） |

> 详见各省份的 .dbf 文件。使用 `dbfread`、`geopandas` 或 QGIS 可直接读取。

## 已知问题

1. **~~坐标系不统一~~**：CRS 已统一为 WGS84 (EPSG:4326)，详见 CHANGELOG.md
2. **字段 Schema 不统一**：各省字段命名和结构差异较大，已建立映射表，详见 SCHEMA_MAPPING.md
3. **浙江村界版本 2 和 3 内容完全相同**（.shp 和 .dbf MD5 一致），已删除版本 3
4. **上海 2 套数据部分字段名被截断**：UTF-8 中文字段名因 DBF 11 字节限制被截断（非 GBK 乱码）。完整字段名已通过字节分析恢复，见 CHANGELOG.md
5. **陕西乡镇界** .dbf 日期显示 1995 年，可能是 DBF 头部日期字段异常，不代表数据实际年份
6. **部分省份含多版本数据**（内蒙、山东、江西、黑龙江、上海），使用时请注意区分

## 使用方法

### QGIS
直接拖入 .shp 文件即可。QGIS 会自动读取 .prj（坐标系）和 .cpg（编码）。

### Python (GeoPandas)
```python
import geopandas as gpd
gdf = gpd.read_file("path/to/file.shp")
print(gdf.crs)  # 查看坐标系
print(gdf.head())  # 查看字段
```

### 格式转换
建议使用 `ogr2ogr` 转换为 GeoPackage（.gpkg）或其他格式：
```bash
ogr2ogr -f GPKG output.gpkg input.shp  # CRS 已统一为 WGS84，无需 -t_srs
```
