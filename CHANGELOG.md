# CHANGELOG — 全国村界数据集预处理记录

## 2026-06-11 — 整理者: @thedavidweng

基于 @elpwc 通过网盘分享的原始数据进行整理。原始数据从网络收集，来源不详，时间跨度约 2010–2022 年（大部分约 2016 年）。

### 清洗操作

| 操作 | 说明 |
|---|---|
| 删除系统文件 | 移除 `.DS_Store`（2 个）、`.lock`（1 个） |
| 解压压缩包 | 黑龙江森工集团下 3 个 RAR + 1 个 ZIP 解压后删除原始压缩包。ZIP 因 GBK 文件名问题使用 `unar` 解决 |
| 修复 .cpg 编码声明 | 陕西乡镇界 .cpg 从 CP936 改为 UTF-8（实际数据为 UTF-8） |
| 创建缺失 .cpg 文件 | 为 15 个缺少 .cpg 的 .dbf 文件补建编码声明（8 个 CP936/GBK，7 个 UTF-8） |
| 删除重复数据 | 浙江村界版本 3（`3浙江村界-备注省市县乡/`）与版本 2 的 .shp 和 .dbf MD5 完全一致，已删除 |
| CRS 声明修正 | 原始 .prj 文件声明了多种 CRS（CGCS2000、Krasovsky_Albers 等），但 .shp 坐标值实际均为 WGS84 地理坐标。已将 .prj 文件统一修正为 WGS84 |
| Schema 分析 | 分析 58 个 .dbf 的 313 个字段名，建立统一映射表（见 SCHEMA_MAPPING.md），覆盖 11 个核心管理层级字段 |

### 未修改的内容

- 未重命名任何字段（字段名保持原始状态）
- 未修改任何 .shp 坐标数据
- 保留了各省多版本/多坐标系数据（内蒙 WGS84+CGCS2000、山东原始+修正版、江西旧+新、黑龙江统计+纠偏、上海 1 套+2 套）

### 上海 2 套数据字段名说明

`上海村界/2上海村社区界-第二套/` 下两个 .dbf 的部分字段名为 UTF-8 中文，因 DBF 字段名 11 字节限制被截断。这些不是 GBK 乱码，而是 UTF-8 三字节序列在第 11 字节处被切断。通过截断模式分析和数据值验证，完整字段名如下：

| 截断的字段名 (hex) | 恢复的完整名称 | 验证依据 |
|---|---|---|
| `e6 89 80 e5 b1 9e e8 a1 97 e9` | 所属街道 | 数据值: "杨行镇" |
| `e6 89 80 e5 b1 9e e5 8c ba e5` | 所属区域 | 数据值: "宝山区" |
| `e5 b1 85 e5 a7 94 e4 bc 9a e4` | 居委会编码 | 数据值: "130934" |
| `e5 b1 85 e5 a7 94 e4 bc 9a e5` | 居委会名称 | 数据值: "2021弄居委" |
| `e6 89 80 e5 b1 9e e5 a4 a7 e5` | 所属大区 | 截断模式推断 |
| `e6 89 80 e5 b1 9e e5 8a 9f e8` | 所属功能区 | 截断模式推断 |
| `e8 a1 97 e9 81 93 e4 bb a3 e7` | 街道代码 | 截断模式推断 |
| `e8 a1 97 e9 81 93 e5 90 8d e7` | 街道名称 | 截断模式推断 |
| `e5 8e 9f e8 a1 97 e9 81 93 e7` | 原街道名 | 截断模式推断 |
| `e5 8e 9f e5 b1 85 e5 a7 94 e4` | 原居委会名 | 截断模式推断 |
| `e5 b1 85 e5 a7 94 e6 a0 b8 e6` | 居委会核实 | 截断模式推断 |
| `e8 87 aa e5 ae 9a e4 b9 89 e7` | 自定义字段 | 截断模式推断 |
| `e7 bd 91 e6 a0 bc e5 90 8d e7` | 网格名称 | 截断模式推断 |
| `e7 bd 91 e6 a0 bc e7 b1 bb e5` | 网格类型 | 截断模式推断 |
| `e7 bd 91 e6 a0 bc e7 bc 96 e7` | 网格编号 | 截断模式推断 |
| `e6 97 a7 e5 b1 85 e5 a7 94 e5` | 旧居委会名 | 截断模式推断 |
| `e8 b4 a3 e4 bb bb e7 bd 91 e6` | 责任网格管理 | 截断模式推断 |
| `e6 9d 91 e5 b1 85 e7 b1 bb e5` | 村居类型 | 截断模式推断 |

> 纠偏版本中含 `ef bf bd`（U+FFFD 替换字符）的字段，说明数据曾被错误转码，WGS84 版本更可靠。字段名存储在 .dbf 头部（固定 11 字节），无法在文件层面修复，完整名称仅记录于此。
