# 中国村境データセット / China Village Boundaries Dataset

**[中文](README.md)** | **[English](README.en.md)** | 日本語

## 概要

中国全土（本土・香港・マカオを含む）の村レベル行政境界の空間データ（ESRI Shapefile 形式）。33 の省級行政区画をカバーし、58 データセット・875,140 レコードを収録しています。

## データ形式

- **主要形式**：ESRI Shapefile（.shp / .shx / .dbf / .prj / .cpg）
- **補助形式**：FileGDB（.gdb）、GeoJSON、KML、GML（一部の省）
- **エンコーディング**：UTF-8（大部分）、GBK/CP936 の既知の問題あり（下記参照）
- **座標参照系（CRS）**：WGS84（EPSG:4326）— 全 58 データセットで統一済み

## 座標参照系（CRS）

**WGS84（EPSG:4326）に統一済み。** 全 58 の .shp ファイルに地理座標（度）が格納されています。全 .prj ファイルは WGS84 を宣言しています。

> 元の .prj ファイルはさまざまな CRS（CGCS2000、Krasovsky_1940_Albers、Hong_Kong_1980_Grid など）を宣言していましたが、.shp の座標値は常に地理座標でした。詳細は CHANGELOG.md を参照してください。

## フィールドエンコーディングに関する注記

全 58 の .cpg ファイルは **UTF-8** エンコーディングを宣言しています。ほとんどのデータセットはフィールド名・値とも UTF-8 ですが、以下の既知の問題があります：

**混合エンコーディングデータセット**（フィールド名は UTF-8、フィールド値は GBK/CP936）：
- 河北・河南・陕西：.cpg は UTF-8 を宣言しフィールド名は読み取り可能ですが、中国語フィールド値（村名など）は GBK です

> GIS ソフトウェアではフィールド名は正しく表示されますが、中国語フィールド値は文字化けする場合があります。`ogr2ogr --config SHAPE_ENCODING GBK` で GeoPackage に変換すると解決します。

**DBF 11 バイト制限による UTF-8 フィールド名の切り詰め**（複数の省）：
- 広東・上海・山西・四川・甘粛・浙江・福建・黒龍江などの一部 .dbf ファイルで、DBF 形式のフィールド名 11 バイト制限により UTF-8 の中国語フィールド名が切り詰められています
- 完全なフィールド名はバイトレベル分析により復元済み — CHANGELOG.md を参照

## フィールドスキーマ

各省の .dbf でフィールド構造が異なります。完全なフィールドマッピングテーブルは **[SCHEMA_MAPPING.md](SCHEMA_MAPPING.md)** に記載（dry run 検証結果を含む）。

マッピングは 11 のコア行政階層フィールドをカバーしています：

| 標準フィールド | 説明 | カバレッジ |
|---|---|---|
| `province_code` / `province_name` | 省級 | 100% |
| `city_code` / `city_name` | 地級市 | 100% |
| `county_code` / `county_name` | 県級 | 100% |
| `township_code` / `township_name` | 郷鎮級 | 100% |
| `village_code` / `village_name` | 村級 | 100% |
| `note` | 備考 | 100% |

> 52/58 データセットが完全にマッピングされています（90%）。残り 6 件は上海の社区管理補助フィールドにおける UTF-8 中国語フィールド名の切り詰めです。完全なフィールド名はバイト分析により復元済み — CHANGELOG.md を参照。

一般的なフィールド名パターン（元の名称）：

| フィールド種別 | 一般的な元の名称 | 説明 |
|---|---|---|
| 行政区画コード | `XZQDM`, `xzqdm`, `AREA_CODE`, `PAC`, `code` | 12 桁または 9 桁の行政区画コード |
| 行政区画名称 | `XZQMC`, `NAME`, `name`, `村名称`, `村级名` | 村・社区名 |
| 階層コード | `省级码`, `市代码`, `区县码`, `乡镇码`, `村级码` | 各レベルの行政区画コード |
| 階層名称 | `省级名`, `市名`, `县区名`, `乡镇名`, `村名` | 各レベルの行政区画名称 |
| 幾何属性 | `SHAPE_Leng`, `SHAPE_Area` | 周長と面積 |

> 各省の .dbf ファイルを直接参照できます。`dbfread`、`geopandas`、QGIS で直接読み取り可能です。

## 既知の問題

1. **~~座標系が統一されていない~~**：CRS は WGS84（EPSG:4326）に統一済み — CHANGELOG.md を参照
2. **フィールドスキーマが統一されていない**：各省でフィールド命名・構造が異なる — マッピングテーブルを整備済み — SCHEMA_MAPPING.md を参照
3. **浙江 v3 を削除**：v2 と v3 の .shp および .dbf が同一（MD5 一致）のため v3 を削除
4. **上海データセットのフィールド名切り詰め**：DBF 11 バイト制限による UTF-8 中国語フィールド名の切り詰め（GBK 文字化けではない）。バイト分析により完全なフィールド名を復元済み — CHANGELOG.md を参照
5. **陝西郷鎮界 .dbf の日付が 1995 年を表示**：DBF ヘッダーの日付フィールド異常の可能性が高く、データの実際の年数を示すものではない
6. **一部の省に複数バージョンが存在**（内モンゴル・山東・江西・黒龍江・上海・浙江）
7. **新疆生産建設兵団は未カバー**：現在の `新疆村界` には新疆ウイグル自治区のデータのみが含まれ、兵団関連の境界は含まれていない

## 使用方法

### QGIS
.shp ファイルを直接ドラッグ＆ドロップするだけです。QGIS が .prj（座標系）と .cpg（エンコーディング）を自動的に読み取ります。

### Python（GeoPandas）
```python
import geopandas as gpd
gdf = gpd.read_file("path/to/file.shp")
print(gdf.crs)   # CRS の確認
print(gdf.head()) # フィールドのプレビュー
```

### 形式変換
`ogr2ogr` を使用して GeoPackage（.gpkg）または他の形式に変換します：
```bash
ogr2ogr -f GPKG output.gpkg input.shp  # CRS はすでに WGS84 のため -t_srs 不要
```
